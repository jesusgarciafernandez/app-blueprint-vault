## Descripción
Habilidad estratégica centrada en la identificación, análisis comparativo y selección óptima de soluciones de software para resolver problemas específicos de negocio u operativos. En un ecosistema saturado de herramientas SaaS, esta skill proporciona un marco de decisión objetivo que trasciende las promesas de marketing. Se basa en el análisis de variables críticas como la curva de aprendizaje, el coste total de propiedad (TCO), la interoperabilidad (vía API/webhooks), la soberanía de datos y la escalabilidad técnica. El objetivo es construir un *stack* tecnológico armonizado que potencie la productividad sin generar silos de información ni deuda técnica innecesaria.

## Cuándo usarla
Escenarios que activan esta skill:
- Durante la fase de diseño de procesos para encontrar la herramienta que mejor se adapte al flujo de trabajo deseado.
- Al detectar ineficiencias o costes excesivos en el *stack* de herramientas actual (auditoría de herramientas).
- Cuando surge una nueva necesidad operativa (ej: implementar un CRM, un sistema de gestión de proyectos o un motor de automatización).
- Para evaluar la viabilidad de migrar de una solución heredada (*legacy*) a una moderna basada en la nube.
- En la fase de consultoría técnica para recomendar soluciones a terceros basadas en requerimientos granulares.

## Requisitos
- Definición clara de los requerimientos funcionales y no funcionales (MoSCoW).
- Acceso a bases de datos de reseñas técnicas y comparativas (G2, Capterra, Reddit).
- Capacidad para entender documentación de API y capacidades de integración.
- Conocimiento básico de modelos de precios (por usuario, por uso, freemium).
- Habilidad para realizar pruebas de concepto (PoC) limitadas.
