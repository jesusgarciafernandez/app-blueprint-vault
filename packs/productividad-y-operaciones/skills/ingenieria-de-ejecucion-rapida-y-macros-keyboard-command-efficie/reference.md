## 5. Instrucciones y Pasos Detallados (Protocolo Maestro)

### Fase 1: Auditoría de Micro-fricciones y Diseño del Mapa de Mandos
**Objetivo:** Identificar los "ladrones de segundos" y centralizar el control.
1.  **Mapeo de la 'Regla de Oro':** IA ayuda a rastrear qué palabras, comandos o clics repite el usuario más de 10 veces al día para convertirlos en Macros.
2.  **Configuración de la 'Tecla Hyper':** Reasignación de teclas poco usadas para acceder a una capa invisible de atajos globales sin conflictos.

**Prompt Maestro de Ingeniería de Ejecución (Command Architect):**
```text
Actúa como un Senior Workflow Optimizer y Experto en Automatización Local. Diseña el mapa de ejecución para: [PERFIL_PROFESIONAL]. 
1. Arquitectura de Lanzador (Raycast/Alfred): Propón 5 'Quick Links' y 5 'Scripts' que ahorren el acceso a la web desde el navegador. 
2. Diccionario de Expansión de Texto: Diseña un sistema de prefijos (Ej: ;m para mail, ;f para firmas, ;d para fechas) que sea intuitivo y rápido. 
3. Diseño de Macro Multi-paso: Crea la lógica para una macro que: [DESCRIBE_FLUJO_REPETITIVO] (Ej: Limpiar el escritorio, abrir Jira, y loguear tiempo). 
4. Gestión Espacial de Ventanas: Define los atajos para mover ventanas a mitades, tercios o monitores externos sin usar el ratón. 
5. Protocolo de Aprendizaje (Muscle Memory): Diseña una rutina de 5 minutos diarios para interiorizar 3 nuevos atajos críticos por semana.
```

## 6. Arquitectura de Automatización Lógica (Agnostic Flow)
*Lógica de respuesta instantánea.*

1.  **Trigger:** El usuario detecta una necesidad operativa (abrir algo, escribir algo, procesar algo).
2.  **Nodo de Ejecución de Comando (Lanzador):** El usuario teclea un comando corto o atajo que dispara una acción o un menú de opciones filtrado por IA.
3.  **Nodo de Expansión/Procesamiento:** Si es texto, se expande; si es una macro, se ejecutan secuencialmente las pulsaciones de teclas y clics virtuales.
4.  **Nodo de Notificación Silenciosa:** El sistema confirma la ejecución (o pide un parámetro extra) mediante una notificación sutil tipo 'Overlay'.
5.  **Output:** Acción completada en milisegundos; flujo de trabajo ininterrumpido; ahorro acumulado de tiempo y energía mental.

## 7. Ejemplo Práctico: El Arquitecto 'Blitz'
**Reto:** Un arquitecto técnico perdía 1 hora al día abriendo planos, buscando el código de color de la marca, redactando correos de revisión estándar y saltando entre AutoCAD y Excel. Sus manos se resentían por el uso excesivo del ratón.
**Acción v2.0:** Implementó Skill 263. Compró un Stream Deck para macros hardware y configuró Raycast con scripts python para buscar en su base de datos de materiales. Creó snippets para todas sus cláusulas legales de correo.
**Resultado:** Ha reducido el uso del ratón en un 70%. Sus revisiones de planos son un 40% más rápidas. El dolor de muñeca ha desaparecido y ahora termina sus informes técnicos en la mitad de tiempo, permitiéndole centrarse en el diseño arquitectónico puro.
