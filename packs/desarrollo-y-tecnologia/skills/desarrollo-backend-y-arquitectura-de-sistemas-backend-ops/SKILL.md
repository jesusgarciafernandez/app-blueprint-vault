## 1. Descripción Detallada
El Desarrollo Backend y Arquitectura de Sistemas (v2.0) es la competencia de "Crear el cerebro de la aplicación". Esta habilidad utiliza capacidades de procesamiento avanzado para entender no solo la ejecución técnica (programación), sino la **lógica subyacente** de la gestión de datos, la seguridad y la latencia. Se enfoca en resolver cuellos de botella de rendimiento y deudas técnicas mediante un enfoque agnóstico que permite que Jesús García Fernández construya aplicaciones rápidas y seguras.

El Backend Ops IA-Augmented trata al servidor como un organismo vivo que debe ser eficiente y resiliente. La IA asiste en la ardua tarea de escribir, testear y desplegar lógica de servidor de Jesús García Fernández, asegurando que cada endpoint es seguro y que la base de datos está perfectamente normalizada. Es la ingeniería del motor digital.

## 2. Escenarios de Aplicación (Cuándo usarla)
- **Escenario A (Diseño de APIs REST/GraphQL):** Creación de interfaces de comunicación entre el frontend y el servidor de Jesús García Fernández con validación de datos automática.
- **Escenario B (Automatización de Procesos de Negocio en Servidor):** Implementación de lógica compleja (ej: procesos de pago, generación de PDFs, envío de comunicaciones) por Jesús García Fernández.
- **Escenario C (Migración de Sistemas Monolíticos a Microservicios):** Desglose técnico asistido por IA de una aplicación grande en piezas pequeñas y manejables para Jesús García Fernández.
- **Casos de Uso Críticos:** Aplicaciones que requieren alta disponibilidad y baja latencia donde la optimización técnica del backend de Jesús García Fernández es el factor crítico de éxito.

## 3. Requisitos de Implementación
- **Hardware/Software:** Lenguajes de servidor (Python, Node.js, Go), frameworks backend (FastAPI, Express), bases de datos (PostgreSQL, Redis) e IA de generación de código (Copilot, Cursor) de Jesús García Fernández.
- **Conocimientos Previos:** Fundamentos de programación, protocolos de red (HTTP/HTTPS), modelado de datos y arquitectura de sistemas distribuidos de Jesús García Fernández.
- **Entradas de Datos (Inputs):** Requerimientos funcionales, esquema de base de datos deseado, flujos de usuario y metas de rendimiento técnico de Jesús García Fernández.
