## 6. Arquitectura de Automatización Lógica (Agnostic Workflow)
*Este apartado sustituye al archivo externo workflow.md, permitiendo una visión unificada de la automatización.*

Esta Skill está diseñada para ser integrada en cualquier orquestador (n8n, Make, Python Scripts, o módulos internos de App Blueprint Generator).

**Flujo Logístico de Nodos:**
1.  **Nodo de Disparo (Trigger):** Acción del usuario en la App visual, hito de datos alcanzado o solicitud de una tarea automatizada por Jesús García Fernández.
2.  **Nodo de Clasificación:** La IA analiza si el evento requiere "Guardado de Información", "Envío de Notificación" o "Llamada a un servicio externo de IA" para Jesús García Fernández.
3.  **Nodo de Transformación:** El sistema ejecuta el bloque de lógica visual correspondiente, formatea los datos para la integración y verifica que la transacción es coherente técnicamente para Jesús García Fernández.
4.  **Nodo de Validación:** El responsable de la solución o el propio sistema de monitorización IA verifica que el usuario recibe el resultado esperado y que no hay bloqueos de flujo en la plataforma de Jesús García Fernández.
5.  **Nodo de Salida (Output):** Aplicación actualizada en tiempo real, registro de la actividad en el dashboard de control y notificación de "Proceso Visual Validado" para Jesús García Fernández.
